# CodexWork

Единая рабочая папка для продолжения задач Codex между Windows и Mac.

## Как пользоваться

1. Храни эту папку внутри облака: OneDrive, Google Drive, Dropbox или другой синхронизируемой папки.
2. Перед началом работы открой `context/current-work.md`.
3. После важной сессии обнови `context/current-work.md` и, если подключен GitHub, запусти `scripts/sync-push.ps1` на Windows или `scripts/sync-push.sh` на Mac.
4. На втором компьютере дождись синхронизации облака, затем запусти `scripts/sync-pull.ps1` или `scripts/sync-pull.sh`.

## Структура

- `projects/` - рабочие проекты, код, скрипты, аналитика.
- `data/` - выгрузки AmoCRM, CRM, CSV, JSON, Excel и другие данные.
- `documents/` - документы, PDF, таблицы, материалы для анализа.
- `context/` - текущий контекст работы Codex и передача задач между компьютерами.
- `secrets/` - локальные `.env` и другие файлы доступа.
- `scripts/` - быстрые команды синхронизации.
- `archive/` - старые выгрузки и завершенные материалы.

## GitHub и облако

Облако синхронизирует всю папку между компьютерами. GitHub нужен для версий кода, заметок и контекста.

По умолчанию `.gitignore` не исключает `.env`, потому что папка предполагается личной. Если позже решишь делать публичные или командные репозитории, это правило нужно пересмотреть.
